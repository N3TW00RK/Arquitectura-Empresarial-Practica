# Práctica de Arquitectura Empresarial

### Documentación de API: REST vs SOAP
Para este proyecto se seleccionó el estilo arquitectónico **REST**. 
* **Justificación:** Se eligió por su ligereza, escalabilidad y el uso de JSON sobre HTTP, lo que permite una mayor interoperabilidad en servicios web modernos comparado con la rigidez de SOAP.
* **Nuevas Tendencias:** Se investigó la automatización mediante **GitHub Actions** para el despliegue continuo de servicios distribuidos.
